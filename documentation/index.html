Introduction

Material is a fully responsive admin web App built with Bootstrap & ReactJS. It uses Sass CSS which makes it easy to modify.

How to Run the Demo?

Note: dist/ folder contains the built distribution (production) version, which is for demo purpose only

You need to put the files on a server to run it. Here we told about how to build a server locally.

You can build a locally server using WAMP or MAMP (if you uses PHP). Also it's pretty easy to build one with Python or Ruby (if you prefer Python or Ruby). And here we'll build a local server with NodeJS

Preparation:

You'll need to install Node.js >=v6.5.0 (NPM comes along with it) and node-static (a NPM package)


Steps:

It's easy if you are familiar with Node. Download Node.js here then install it, and install node-static package with one command line

$ npm install -g node-static

The usage of node-static is simple. Just enter the "dist" directory with command line

$ cd dist/

and type:

$ static

It'll return something like this: 

serving "." at http://127.0.0.1:8080

Then you can view the template on http://127.0.0.1:8080 with your browser

Note: Because the dist/ folder built with webpack uses absolute path, you need to run it from root path, if not, please compile a new dist/ with a new publicPath variable and run `npm run build`.

How to Use It?

In order to do development with it, first you'll need have the following tools installed:

These tools are for development purpose only, you don't need them in production environment, meaning you can use any back-end languages you like, NodeJS is just one of the options, not required

  • Node.js >=v6.5.0 (NPM comes along with it), just download and install it, that's it.
  • Ruby (if you are using a Mac, you already have it), Sass for Scss CSS.

    • A few steps to get started:

    1. Enter "material" folder with your command line tool.

    2. Install all the NPM packages with command: 

    $ cd material
    $ npm install

    3. Then type npm start and you are ready to go :)

    $ npm start

    4. It will automatically open the web app with your default browser


    Note: All the source code for development are inside ./material/src/ folder.

    Layouts & Color Schemes

    You can simply update a few variables to change the default settings.

    Open material/src/constants/Config.js file, you'll find following code. You can change the value to update the default settings.

    const APPCONFIG = {
    brand: 'Material',
    user: 'Lisa',
    year: year,
    productLink: 'https://themeforest.net/user/arousing/portfolio',
    AutoCloseMobileNav: true,                           // true, false. Automatically close sidenav on route change (Mobile only)
    color: {
        primary:    '#00BCD4',
        success:    '#8BC34A',
        info:       '#66BB6A',
        infoAlt:    '#7E57C2',
        warning:    '#FFCA28',
        danger:     '#F44336',
        text:       '#3D4051',
        gray:       '#EDF0F1'
    },
    settings: {
        layoutBoxed: false,                             // true, false
        navCollapsed: false,                            // true, false
        navBehind: false,                               // true, false
        fixedHeader: true,                              // true, false
        sidebarWidth: 'middle',                         // small, middle, large 
        colorOption: '32',                              // String: 11,12,13,14,15,16; 21,22,23,24,25,26; 31,32,33,34,35,36
        theme: 'light',                                 // light, gray, dark
    }
}
